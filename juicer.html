1.我们为什么需要模板引擎 开发中，我们经常需要发起Ajax请求,向服务端提供的接口请求数据，然后解析数据并在页面中显示出来。徒手写代码时，必然会写大量混杂html与js的代码，不仅拼接字符串影响性能，同时写代码也很慢，最重要的是容易出错。使用前端模板引擎，就能比较好的解决这个问题，那就是数据与结构分离，简化代码编写，减少出错。除此之外，模板引擎还能帮助我们进行数据处理与逻辑分离、数据表现与逻辑分离，增加代码重用率。 2.模板引擎的本质 所谓模板引擎，不过是一个编译器的前端，它规定一套语法规则，开发中通过这套语法规则的辅助下实现业务上的功能，然后，通过这个编译器将这段基于既定语法规则写成的代码转换为目标代码。就Juicer而言，混合了{}及HTML的就是源代码，或者是被称为模板，而目标代码就是模板经过引擎处理后的得到的符合W3C标准的HTML。 模版引擎 juicer 当前最新版本: 0.6.14，Juicer 是一个高效、轻量的前端 (Javascript) 模板引擎，使用 Juicer 可以是你的代码实现数据和视图模型的分离(MVC)。除此之外，它还可以在 Node.js 环境中运行，Juicer 代码完全开源并托管在 Github 上了。 1 .名字的由来 榨汁机 倘若我们把数据比作新鲜可口的水果，把模板看做是水，Juicer 就是把水果和水榨出我们需要的HTML代码片段的榨汁机。 2.Juicer 的引入 Javascript 代码： var tpl = 'Hi, {@include "#subTpl", subData}, End.'; juicer(tpl, { subData: { name: 'juicer' } }); //输出 Hi, I'm sub content, juicer, End. //或者通过数据引入子模板，下述代码也将会有相同的渲染结果： var tpl = 'Hi, {@include subTpl, subData}, End.'; juicer(tpl, { subTpl: "I'm sub content, ${name}", subData: { name: 'juicer' } }); * 在 Node.js 环境中运行 在命令行中执行:npm install juicer 在代码中这么引入: var juicer = require('juicer'); var html = juicer(tpl, data); * 一个完整的例子 HTML 代码: Javascript 代码: var data = { list: [ {name:' guokai', show: true}, {name:' benben', show: false}, {name:' dierbaby', show: true} ], blah: [ {num: 1}, {num: 2}, {num: 3, inner:[ {'time': '15:00'}, {'time': '16:00'}, {'time': '17:00'}, {'time': '18:00'} ]}, {num: 4} ] }; var tpl = document.getElementById('tpl').innerHTML; var html = juicer(tpl, data);
