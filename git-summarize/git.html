Git 1.Git 是一种分布式版本控制系统，即指的是没有中心服务器，每个人的机器都是一个完整的库，每个人都可以使用分享自己的代码 ，或者fork其他人的代码，这种分布式管理其实有很多优点，一、可以提供备份，有记录多个版本的功能，在我们代码出错时 ，可以查看历史操作，可以进行版本回滚，二，自动合并，可以解决多人开发冲突的问题。 2.在git中，我们将需要进行版本控制的文件目录叫做一个仓库，每个仓库可以理解成一个目录，这个目录里的所有文件都是通过git控制的，git 会记录每次提交时的信息。 3.说一下开发流程吧 一、首先要在gitHub上创建远程仓库用来存储项目，首先要让远程仓库和本地有所关联，可以通过git clone使远程仓库和本地达成信任关系，git init 初始化 ，git pull origin master本地才有master分支 ，git 的基本构建完成。 二、下面说下git的常用命令 用git 操纵本地仓库 git init :创建一个本地仓库 git add read.txt:把本地文件（工作区）添加到缓存区 git add .把本地所有文件全部添加到缓存区 git commit –m “注释”把缓存区的所有文件都添加到仓库中 git status查看git仓库当前的状态 git diff read.txt: 查看文件具体修改了哪里 git log：查看最近到最远提交到仓库的文件信息（一串数字为特有的时间序列id 可以根据它进行版本前后回滚） git reset --hard HEAD^： 回退到上一次 commit的时候 git reset --hard bdeacd: 版本前进 只能根据 id进行前进（id只写出前六位就好了） git checkout -- read.txt: 工作区恢复到最近一次的commit或者add git branch dev: 创建dev分支 git checkout dev： 切换到dev分支上开发 git branch：查看所有分支 git branh -d dev：删除分支 （-D 强制删除） git merge dev: 合并分支 三、如果开发的过程中发现主分支master上有bug但正在开发的dev分支没有开发完怎么办，切记一点不要在master分支上直接开发，保存dev工作现场回到master上创建解决bug的分支如issue分支，解决完回到master分支合并再回到dev 恢复现场 git stash 保存工作现场 git pop 恢复工作现场
